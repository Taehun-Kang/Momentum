# 6. 프론트엔드 구현 요약

> 상세 문서: [docs/basic/6.프론트엔드 구현 상세.md](../basic/6.프론트엔드%20구현%20상세.md)

## 기술 선택
- **Vanilla JS**: 프레임워크 오버헤드 없음, 빠른 로딩
- **Component 클래스**: 재사용 가능한 컴포넌트 시스템
- **SPA Router**: 자체 구현
- **PWA**: 오프라인 지원, 설치 가능

## 프로젝트 구조
```
frontend/
├── js/
│   ├── app.js          # 메인 진입점
│   ├── router.js       # SPA 라우터
│   ├── core/           # 핵심 기능 (api, auth, state)
│   ├── components/     # 재사용 가능한 컴포넌트
│   ├── pages/          # 페이지 컴포넌트
│   └── services/       # 비즈니스 로직
├── css/                # 스타일시트
└── index.html          # SPA HTML
```

## 핵심 컴포넌트
- `VideoCard`: 영상 카드 UI
- `ChatInterface`: AI 채팅 UI
- `TrendingSection`: 트렌드 섹션
- `InfiniteScroll`: 무한 스크롤

## 성능 최적화
- 번들 크기 < 100KB
- LCP < 2.5초
- 코드 스플리팅
- 이미지 lazy loading
